# OCR実装方針の変更履歴

**日付:** 2026-05-21

## 概要

セキュリティの強化と会社の制約内での運用を実現するため、OCRの実装方針をクラウドベースの「Google Cloud Vision API」から、クライアントサイドで動作する「**Tesseract.js**」ライブラリへ正式に変更しました。

## 決定事項と理由

当初検討していた、フロントエンドから直接Google Cloud Vision APIを呼び出す案は、主に以下の2つの懸念点により採用を見送りました。

1.  **高いセキュリティリスク:** 従量課金が発生するAPIキーをブラウザクライアントに直接露出させることは、重大なセキュリティ脆弱性となり得ます。HTTPリファラー制限のような対策は存在するものの、完全ではなく、不正利用や予期せぬ料金が発生するリスクを排除できません。

2.  **会社の請求・支払いに関する制約:** APIキーを隠蔽するための安全な代替案（Cloud Functionsなどをバックエンドのプロキシとして利用する手法）は、実現不可能でした。これは、必要なネットワーク機能を有効化するにはFirebaseの「Blazeプラン」へのアップグレードが必須となりますが、会社のポリシーとしてクレジットカードの事前登録が許可されていないためです。

これらの制約を考慮した結果、**Tesseract.js** が最適な解決策として選択されました。

### Tesseract.jsの利点:

-   **バックエンド/APIキーが不要:** ユーザーのブラウザ内で完全に動作するため、OCR処理のためのバックエンド、ネットワーク通信、APIキーが一切不要になります。
-   **セキュリティの向上:** 機密データや画像がネットワーク経由で送信されることがないため、プライバシーとセキュリティが大幅に向上します。
-   **コスト不要:** 無料で利用できるオープンソースライブラリです。

## アーキテクチャへの影響

-   Express.jsサーバー（`backend/server.js`）および、その他全てのバックエンド関連インフラは、本プロジェクトにおいて不要（廃止）と見なされ、プロジェクトから削除されます。
-   本プロジェクトは、Firebase（Auth, Firestore）とTesseract.jsがブラウザで直接動作する、純粋なクライアントサイドレンダリング（CSR）アプリケーションとして運用されます。
