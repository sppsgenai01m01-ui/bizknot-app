# BizKnot 課題・技術的負債バックログ

本ドキュメントは、プロジェクト開発進行において一時的に回避・妥協した実装や、将来のフェーズ（本稼働時など）で必ず解消すべき技術的課題を記録するものです。

---

## 1. Firebase Storage（バケット）への画像保存の移行

*   **課題の内容:**
    現状（2026/05/21時点）、Google Cloudの仕様変更により無料枠（Sparkプラン）で新規バケットが作成できなくなりました。
    そのためMVPデモを無料枠で完了させるための緊急避難的措置として、**「画像をCanvasでBase64文字列に圧縮し、Firestoreのドキュメント内に直接保存する」**というアーキテクチャに変更しています。

*   **なぜ解消すべきか（技術的負債の理由）:**
    Firestoreの1ドキュメントあたりのサイズ制限は最大1MBです。現状は画質を落としてリサイズすることで100KB前後に収めていますが、将来的に「高画質な原本名刺画像を保存したい」といった要件が出た場合、Firestoreの容量制限によりエラーとなります。また、データベース側に巨大な文字列が保存されると、将来的なクエリやバックアップのパフォーマンス低下を招く恐れがあります。

*   **解消するためのステップ:**
    1. デモ完了後、プロジェクトを「Blazeプラン（従量課金）」にアップグレードし、Firebase Storage を有効化する。
    2. GCPコンソールからバケットのCORS設定（`cors.json`の適用）を行う。
    3. `public/creation.js` にコメントアウトして残してある「Storageへのアップロード処理（`storageRef.put()`）」のコードのコメントを外し、復活させる。
    4. Base64圧縮処理を削除し、本来の `imageUrl`（StorageのダウンロードURL）をFirestoreに保存するように元に戻す。

*   **対応予定時期:**
    デモ公開後、本稼働に向けたインフラ整備フェーズにて実施。

## 2. OCR失敗・名刺画像認識不可時のエラーハンドリングとリカバリー導線の改善

*   **課題の内容:**
    現状、名刺画像として認識できなかった場合、または画像データのOCRが不十分であった場合に、「画像がありません 画像を見ながら右側のフォームを修正してください」というメッセージが表示される。
    しかし、これには以下の2点の問題（課題）がある。
    1. **リカバリー導線の不足**: 撮影に失敗した時や誤ったファイルを選択した際に、その画面で即座に「撮り直す」または「ファイルを選択し直す」ためのボタン（アクション）が存在しない。
    2. **メッセージとレイアウトの不整合**: メッセージ内に「右側のフォーム」と記載されているが、実際にはPC版・スマホ版を問わず、画像の下にフォームが配置されるレイアウトになっているため、方向の表現が現状のUI設計と合致していない。

*   **なぜ解消すべきか（UX向上の理由）:**
    撮影のブレや光の反射などによりOCRに失敗することは実運用上必ず発生する。その際、一度登録フロー全体をキャンセルして最初からやり直す必要があり、UXの著しい低下を招く。また、文言の不整合（「右側」）はデモや本稼働時の製品クオリティ（信頼感）を損なう要因となる。

*   **解消するための仕様（提案仕様）:**
    *   **スマホ撮影ルート時**:
        「名刺が正しく読み込めませんでした。もう一度名刺を枠内に収めて撮影し直してください。」等のメッセージと、**「名刺を撮り直す」ボタン**（ネイティブカメラを再起動するボタン）を画面内に表示する。
    *   **ファイル/アルバム選択ルート時（PC/スマホ共通）**:
        「名刺が正しく読み込めませんでした。文字がはっきりと写っている別の画像ファイルを選択してください。」等のメッセージと、**「画像ファイルを選択し直す」ボタン**を表示する。
    *   **文言の修正**:
        案内文言を「画像を見ながら、**下記のフォーム**で内容を確認・修正してください。」等、レイアウトに依存しない表現へ修正する。

*   **対応予定時期:**
    将来のUXブラッシュアップフェーズまたは本稼働前のUI微調整フェーズ。
