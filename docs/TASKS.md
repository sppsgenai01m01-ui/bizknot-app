# 開発タスク管理 & 技術仕様メモ

このファイルは、プロジェクトの将来的な課題、実装予定の機能、技術的な仕様メモなどを一元管理するためのものです。

## 1. 【将来実装】検索機能の高度化

- **課題:** 現在の検索はFirestoreの制約により**前方一致**のみ。部分一致やあいまい検索ができない。
- **方針:** 主要機能のリリース後、本格的な検索エンジン（Algolia等）の導入を検討する。

## 2. 【着手】AIによる名刺OCR機能

- **目的:** 名刺画像をアップロードし、テキストを自動でフォームに入力することで、ユーザーの入力負荷を劇的に削減する。

- **技術選定の経緯:**
    - **当初案 (Cloud Functions):** 当初はFirebase Cloud Functions + Vision APIで実装予定でした。しかし、Cloud Functionsの無料プランでは外部APIへのリクエストが制限されており、Vision APIの利用には有料プランへのアップグレードが必須でした。
    - **断念理由:** プロジェクトの都合上、**クレジットカード情報の登録が必要な有料プランへの移行を断念**したため、この機能の実装は見送られていました。
    - **現在の方針 (Render):** 現在のバックエンドである**Renderは、外部APIへのリクエスト制限がありません**。そのため、Vision APIの無料枠（1,000枚/月）を活用したOCR機能の実装が可能となっています。

- **実装アーキテクチャ (Render版):**
    1. **フロントエンド:** ユーザーがアップロードした画像データを、Render上のバックエンドサーバーに送信する。
    2. **バックエンド (Render):**
        - 画像データを受け取るAPIエンドポイント (`/api/ocr`) を作成する。
        - サーバー側でVision APIを呼び出し、画像からテキストを抽出する。
        - 抽出結果を整形し、フロントエンドにJSON形式で返却する。
    3. **フロントエンド:** バックエンドから受け取ったJSONデータを使い、登録フォームの各項目を自動入力する。

## 3. データ管理の仕様

### 3.1. 論理削除 <font color="green">(実装済み)</font>

- **仕様:** ユーザーがデータを削除した際、レコードを物理的に削除せず、`isDeleted: true` のフラグを立てる。
- **目的:** 誤操作からの復旧パス確保と、監査ログ用の履歴保持。
