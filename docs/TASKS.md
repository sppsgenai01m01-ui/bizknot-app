## 1. 【最重要】検索機能の高度化（プランA）

- **現状の課題：**
  - 現在の検索機能（プランB）は、Firestoreのデータ構造の工夫による簡易的な部分一致検索である。
  - この方法では、`searchKeywords`フィールドがドキュメントサイズを肥大化させる可能性があり、また「式会」のような単語の途中からの検索には対応できない。

- **解決策（プランA）：**
  - 本格的な検索エンジンである **Algolia** を導入する。

- **実装の概要：**
  1.  Algoliaの無料アカウントを契約し、初期設定を行う。
  2.  バックエンド(`backend/server.js`)に、Firestoreのデータが更新されるたびに、自動でAlgoliaにもデータを同期させる処理を追加する。（Cloud FunctionsやWebhookを利用）
  3.  フロントエンド(`public/list.js`)の検索処理を、Firestoreへの問い合わせから、Algoliaへの問い合わせに切り替える。
  4.  これにより、タイポ（打ち間違い）許容、関連度順ソートなど、快適で高精度な検索が実現できる。

- **着手タイミング：**
  - アプリケーションの主要機能が一通り実装完了した後。

## 2. AIによる名刺画像の自動文字認識（OCR）機能

- **現状の課題：**
  - コストとデプロイの制約（Firebase無料プランの仕様）により、Cloud Functionsを利用したバックエンド処理を意図的に削除している。
  - そのため、名刺登録はユーザーによる完全な手動入力となっている。

- **解決策：**
  - プロジェクトが収益化し、FirebaseのBlazeプラン（有料プラン）へ移行した際に、Cloud FunctionsとGoogle Cloud Vision APIを利用したOCR機能を復活させる。

- **実装の概要:**
  1.  FirebaseプロジェクトをBlazeプランにアップグレードする。
  2.  削除した`functions`ディレクトリを復活させ、Cloud Functionsを再デプロイする。
  3.  フロントエンド(`creation.js`)から、アップロードされた画像をCloud Functionに渡す処理を復活させる。
  4.  Cloud Function内で、`@google-cloud/vision` APIを呼び出し、画像からテキストを抽出する。
  5.  抽出したテキストをフロントエンドに返し、各フォームに自動入力する。

- **備考：**
  - この機能こそが、このアプリケーションの価値を大きく飛躍させるキラー機能である。
