# BizKnot プロジェクト備忘録 (Antigravity Memory)

## プロジェクト概要
- **プロジェクト名**: BizKnot (名刺管理Webアプリケーション)
- **アーキテクチャ**: CSR (Client-Side Rendering) 
- **技術スタック**: Vanilla JS (ES6+), HTML5, TailwindCSS (CDN), Firebase (Auth, Firestore, Hosting, Storage)
- **インフラ環境**: Firebase Studio (Project IDX)
- **制約事項**: 会社アカウントのため、クレジットカードの事前登録が不可（Firebase BlazeプランやGCPの従量課金APIが利用できない）。

## 本日の作業と決定事項 (2026/05/21)

### 1. 開発環境のトラブルシューティング
- **事象**: `node server.js` 実行時に `EADDRINUSE` やタイムアウトが発生する。
- **解決**: 現在のCSRアーキテクチャにおいて `server.js` などのバックエンドは完全に不要（過去の遺物）であると結論付けた。また、Project IDX環境では自動的にWeb Preview用サーバーが立ち上がるため、自前でのサーバー手動起動自体が不要。

### 2. OCR機能の実装方針の大幅変更
- **当初の案**: フロントエンドから直接 Google Cloud Vision API を叩く。
- **課題**: APIキーの露出によるセキュリティリスクと、クレカ登録不可の制約によるバックエンド(Cloud Functions)構築不可の壁。
- **新方針**: APIキーや外部バックエンド通信を一切必要としない、完全無料・ブラウザ内で完結する安全なOCRライブラリ **「Tesseract.js」** を採用。

### 3. Tesseract.js を用いた実装とUX設計
- `public/business_card_creation.html` に不足していたFirebase SDKとTesseract.jsの読み込みを追加。
- `public/creation.js` に画像アップロード時のOCR解析ロジックを追加。
- **UXフェイルセーフ**: Tesseract.jsの日本語精度課題をカバーするため、メール・電話番号・会社名（キーワードベース）のみを正規表現で自動抽出し、**読み取った全文（生データ）を必ず「メモ欄」に保存する仕様**とした。これにより、次画面のフォームでユーザーが簡単にコピペできる状態を担保し、ユーザーの入力ストレスを最小化。

## 本日の作業と決定事項 (2026/05/22)

### 1. OCR精度の抜本的改善（テストエンジニア視点でのチューニング）
- **パラメーター変更**: `jpn+eng` マルチ言語ロード、`tessedit_pageseg_mode (PSM): 11`（SPARSE_TEXT）を採用し、散在する文字や記号の認識を強化。
- **画像の前処理**: 当初導入した「独自の二値化（白黒化）」は、スマホ撮影時の影による黒つぶれを引き起こすため完全廃止。カラー画像を1200pxの高解像度でTesseractへ渡すよう変更。
- **補完ロジックの実装**: `zipcloud` APIによる郵便番号からの住所補完、レーベンシュタイン距離を用いた「役職」のファジーマッチによる誤字補正を追加。
- **登録フローの改善**: OCR直後の即時Firestore保存を廃止し、`sessionStorage` に一時保存してから手動確認画面へ遷移させる仕様へ変更（ゴミデータの蓄積防止）。

### 2. ユーザー要望による新機能（UX向上）の追加実装
- **名刺一覧**: `IntersectionObserver` を用いた無限スクロール（ページネーション）と、BOM付きUTF-8での「CSVエクスポート」機能を実装（スマホ表示時はCSVボタンを非表示）。
- **ダッシュボード**: 「総登録数」「今月の登録数」を動的に集計して表示するよう改修。
- **名刺詳細**: 
  - 画像クリック時の拡大表示（ライトボックス）機能。
  - 編集前のデータを退避する `history` サブコレクションを導入し、更新履歴のモーダル表示機能を追加。
  - 連絡先へのアクション導線（PC/スマホ共通のメール送信ボタン、スマホ限定の電話発信ボタン）を追加。
  - 「削除ボタン」は、管理者（admin）または登録者本人のみに表示するアクセス制御を実装。

### 3. 未実装画面のアーキテクチャ再定義（無課金対応）
- **制約（Cloud Functions利用不可）** を踏まえ、未実装である「ユーザー管理（SCR-008）」および「監査ログ（SCR-010）」は、すべてフロントエンド完結（Firestoreベース）で実装するよう仕様を再定義。
- **ユーザー管理**: ログイン時に自動で `users` コレクションを作成し、管理者が画面上から Role (ADMIN/USER) や Status (ACTIVE/SUSPENDED) を切り替える論理アクセス制御（RBAC）を採用。
- **監査ログ**: サーバーサイドでのセキュアなロギングを諦め、JavaScriptから重要アクションごとに `auditLogs` コレクションへ書き込む方式を採用。
- 上記の仕様変更を `docs/BizKnot_追加・変更仕様定義.md` に反映。

### 4. ユーザー管理・監査ログ画面の完全実装とバグ修正（午後）
- **OCR（透過PNG）の不具合修正**: 透過背景のPNGをアップロードした際、JPEG変換時に背景が黒つぶれしてOCR精度が崩壊するバグを特定。`creation.js` にてCanvasの背景を強制的に白（#FFFFFF）で塗りつぶす処理を追加し解決。
- **認証共通ロジックの統合**: `dashboard.html` などで独自に行われていた認証判定を `app.js` の `protectPage` に統合。これにより初回ログイン時の `users` コレクション自動作成が正常に機能するように修正。
- **ユーザー管理画面 (SCR-008) の実装**: 管理者（admin）のみがアクセス可能。登録ユーザーの一覧表示と、権限（user/admin）・ステータス（ACTIVE/SUSPENDED）の即時変更機能を実装。
- **監査ログ画面 (SCR-010) の実装**: 管理者のみアクセス可能。Firestoreの `auditLogs` コレクションからログ一覧を取得し、CSV形式でエクスポートする機能を実装。ログイン成功時などにログを自動記録する仕組みも追加。
