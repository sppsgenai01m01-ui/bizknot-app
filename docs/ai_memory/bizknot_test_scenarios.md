# BizKnot テストシナリオ（観点）一覧

本ドキュメントは `BizKnot_追加・変更仕様定義.md` および基本設計に基づき、Playwrightを用いたUI自動テスト（内部結合・E2E）を実装するためのテスト観点を箇条書きで洗い出したものです。
特に**「セキュリティリスクの低減」**を最重要視し、権限チェックや異常系に重点を置いています。

---

## 1. 認証・ログイン機能 (SCR-001)

### 正常系
- [ ] 許可されたドメイン（`@spps.co.jp` または `@gmail.com`）のGoogleアカウントでログインできること。
- [ ] 初回ログイン時に、Firestoreの `users` コレクションに初期ステータス（Role: USER, Status: ACTIVE）でデータが自動作成されること。

### 異常系・セキュリティ系
- [ ] 許可されていないドメインのアカウントでログインを試みた場合、エラーとなりログインできないこと。
- [ ] `Status` が `SUSPENDED` に設定されているアカウントでログインを試みた場合、強制ログアウトされ、エラーメッセージが表示されること。
- [ ] ログアウトボタン押下後、ブラウザの「戻る」ボタンで保護された画面（ダッシュボード等）にアクセスできないこと（未ログイン状態でのアクセス弾き）。

---

## 2. 名刺登録・OCR機能 (SCR-003, SCR-004)

### 正常系
- [ ] 名刺画像（JPG/PNG）をアップロードまたはカメラで撮影した際、画像がBase64に変換され、OCR解析が走ること。
- [ ] OCR解析完了後、Firestoreに即時保存されず、解析データと画像が `sessionStorage` に保持された状態で手動入力フォーム（SCR-004）へ遷移すること。
- [ ] フォーム画面にて、左側に読み込んだ元画像（プレビュー）が表示されること。
- [ ] フォーム画面で不足情報を手動入力し「登録」を押下した際、Firestoreにデータが保存され、一覧画面へ遷移すること。

### 異常系
- [ ] 5MB以上の画像、または非対応の拡張子をアップロードした場合、フロントエンドでエラーアラートが出ること。
- [ ] OCRの解析が30秒を超過した場合、タイムアウトとなり、エラーアラートと共に手動入力画面へ遷移すること。
- [ ] フォーム画面で必須項目（会社名、氏名）を空にして登録ボタンを押下した場合、バリデーションエラーとなること。

---

## 3. 名刺一覧・検索機能 (SCR-006)

### 正常系
- [ ] ダッシュボードおよび一覧画面にて、論理削除された名刺（`deletedAt` が存在）が表示されないこと。
- [ ] 画面を下へスクロールした際、IntersectionObserverが発火し、追加の20件がロードされること（無限スクロール）。
- [ ] 検索ボックスに「会社名」や「氏名」の一部を入力した際、フロントエンド（インメモリ）でフィルタリングされ、正しい結果が表示されること。
- [ ] PCブラウザで閲覧時のみ「CSVエクスポート」ボタンが表示され、クリックで検索結果がダウンロードできること。

### 異常系
- [ ] スマホサイズ（モバイルビューポート）で閲覧時、「CSVエクスポート」ボタンが非表示（DOM上で hidden）になっていること。

---

## 4. 名刺詳細・編集・履歴機能 (SCR-005)

### 正常系
- [ ] メールアドレスのリンクをクリックすると、`mailto:` が発火すること。
- [ ] スマホビューポートで閲覧時、電話番号リンクをクリックすると `tel:` が発火すること。
- [ ] 編集画面で名刺データを更新した際、Firestoreの `history` サブコレクションに元データが退避（バージョン管理）されること。
- [ ] 編集完了後、別の画面に遷移せず、トースト風の成功メッセージが表示されること。

### セキュリティ・権限系
- [ ] **【一般ユーザー (USER)】** 自分が登録した（`ownerId` が一致する）名刺の詳細画面を開いた場合、「削除」ボタンが表示されること。
- [ ] **【一般ユーザー (USER)】** 他人が登録した名刺の詳細画面を開いた場合、「削除」ボタンが非表示になっていること。
- [ ] **【管理者 (ADMIN)】** 他人が登録した名刺であっても、詳細画面に「削除」ボタンが表示され、論理削除が実行できること。

---

## 5. ユーザー管理機能 (SCR-008)

### セキュリティ・権限系
- [ ] **【一般ユーザー (USER)】** ダッシュボード等に「管理者メニュー（ユーザー管理）」のボタンが表示されないこと。
- [ ] **【一般ユーザー (USER)】** URLを直接叩いて `user-management.html` にアクセスした場合、強制的にダッシュボードまたはインデックスへリダイレクトされること（アクセス不可）。
- [ ] **【管理者 (ADMIN)】** ユーザー管理画面にアクセスでき、全ユーザーの一覧が表示されること。
- [ ] **【管理者 (ADMIN)】** 特定のユーザーのRole（USER/ADMIN）およびStatus（ACTIVE/SUSPENDED）を変更でき、Firestoreに即時反映されること。

---

## 6. 監査ログ機能 (SCR-010)

### 正常系
- [ ] 以下の操作を行った際、バックグラウンドで `auditLogs` コレクションにログが記録されること。
    - ログイン成功時 (`LOGIN`)
    - 名刺の新規登録 (`CREATE_CARD`)
    - 名刺の更新 (`UPDATE_CARD`)
    - 名刺の削除 (`DELETE_CARD`)

### セキュリティ・権限系
- [ ] **【一般ユーザー (USER)】** ダッシュボード等に「監査ログ」のボタンが表示されないこと。
- [ ] **【一般ユーザー (USER)】** URLを直接叩いて `audit-log.html` にアクセスした場合、強制的にリダイレクトされ弾かれること。
- [ ] **【管理者 (ADMIN)】** 監査ログ画面にアクセスでき、履歴が降順で表示されること。
- [ ] **【管理者 (ADMIN)】** 「CSV出力」ボタンから監査ログのCSVデータがダウンロードできること。
