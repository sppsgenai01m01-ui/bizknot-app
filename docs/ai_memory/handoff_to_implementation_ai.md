# 実装AI向け：テスト自動化に関する申し送り書（絶対遵守事項）

このドキュメントは、本プロジェクトのアーキテクトAI（Antigravity）から実装AIへの「テスト自動化作業に関する絶対遵守のガイドライン」です。
テストコードの実装にあたり、必ず以下の制約と提供されたセットアップファイルを正として作業を進めてください。

---

## 1. アーキテクチャと権限管理の制約（超重要）
本プロジェクトは「完全無課金（Cloud Functions不使用）」のアーキテクチャを採用しています。

*   **【禁止事項】** `firebase-admin` を用いた **Custom Claims（カスタムクレーム）の付与による権限管理は絶対にしないでください。**
*   **【遵守事項】** ユーザーの権限（`admin` / `user`）およびステータス（`ACTIVE` / `SUSPENDED`）は、**Firestoreの `users` コレクション内のドキュメント（`users/{uid}`）で論理的に管理**されています。
*   そのため、事前データセットアップ処理（`global-setup.js`）には、「Authへのユーザー作成」だけでなく「Firestoreの `users` コレクションへのドキュメント登録」の処理を組み込んでいます。**この提供された `global-setup.js` のロジックを改変しないでください。**

## 2. エミュレータとポート競合の解決ルール
E2Eテストは、必ず `Firebase Local Emulator Suite` に対して実行します。

*   **【遵守事項】** エミュレータのデフォルトポート（Firestore: 8081, Auth: 9099, Hosting: 8080）は**変更しないでください**。
*   `firebase.json` のポート番号や `playwright.config.js` の設定を勝手に書き換えると、フロントエンド（`app.js` の `useEmulator` 等）との通信の不整合が起き、テストが永久に失敗します。
*   **【禁止事項】** もし `port taken` (ポート使用中) エラーが出た場合、**ポート番号をずらして解決するアプローチは禁止**です。必ずOSのコマンド（Windowsの場合は `Get-Process -Id (Get-NetTCPConnection -LocalPort 8081).OwningProcess | Stop-Process -Force` 等）を使用して、裏で生き残っているゾンビプロセスをキルして解決してください。

## 3. テスト階層と役割分担
テストの肥大化とメンテナンス崩壊を防ぐため、以下の役割分担に従ってください。

*   **【Playwright (E2E) が担当する領域】**
    *   画面の正常な遷移（ログイン → 名刺一覧 → 登録フォーム 等）
    *   主要なUI要素の描画確認（「管理者メニュー」の表示/非表示など）
    *   基本的なフォーム入力と登録、CSVエクスポート等のユーザーシナリオ
*   **【Playwright で実装してはいけない領域】**
    *   **ロジックの極限テスト:** レーベンシュタイン距離計算や、OCR抽出正規表現の網羅的テストは、後日 Vitest/Jest 等の単体テストで実装します。
    *   **セキュリティルール（権限突破）のテスト:** 「URLを直接叩く」「DevToolsで改ざんする」といったバックエンド寄りのセキュリティテストは、Playwrightではなく `@firebase/rules-unit-testing` を用いてFirestoreセキュリティルールの単体テストとして後日実装します。Playwrightでは「UI上に削除ボタンが出ないこと」などのフロントエンドの挙動確認に留めてください。

## 4. ブランチ操作の禁止
*   **【禁止事項】** 事前の許可なく、Gitで新しいブランチ（`UnitTest` 等）を切ったり、チェックアウトしたりしないでください。必ず現在のワークスペース（ローカルフォルダ）の最新コードを「正」として作業してください。

---
**以上のルールに同意した上で、すでに用意されている `playwright.config.js` と `tests/global-setup.js` を利用し、具体的なテストシナリオ（名刺登録フロー等）のテストコード（`*.spec.js`）の作成を開始してください。**
