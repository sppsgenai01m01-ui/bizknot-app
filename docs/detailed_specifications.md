# 詳細仕様書（補足仕様・エッジケース定義）

本ドキュメントは、単体テスト・セキュリティテストの実装過程で判明・定義された詳細な仕様（エッジケース・異常系の挙動など）をまとめたものです。基本設計書や仕様メモを補完する目的で作成されています。

## 1. OCR解析ロジック（ocrParser.js）の詳細仕様

### 1.1 役職の表記揺れ補正（レーベンシュタイン距離）
- **仕様**: 「代表取締役」という文字列に対して、OCRの読み取り精度による誤字脱字（例：「代表取結役」「代表取締」など）が発生した場合、**レーベンシュタイン距離（編集距離）が2以内**であれば正しく「代表取締役」として自動補正・許容する。

### 1.2 メールアドレスの抽出（境界値処理）
- **仕様**: 名刺上に「Email: test@example.com」と記載されている場合、不要なプレフィックス（Email: 等）を自動で除去し、純粋なアドレス部分のみを抽出する。

### 1.3 電話番号・FAX・携帯電話の振り分けロジック
- **仕様**: 抽出した電話番号フォーマットに対し、以下の優先順位とコンテキスト（文脈）で種別を判定する。
  1. **FAXの判定**: プレフィックスに「fax」が含まれる場合は `fax` に割り当てる。
  2. **携帯電話の判定**: プレフィックスに「携帯」「mobile」が含まれる、または番号自体が `090`, `080`, `070` から始まる場合は `mobilePhone` に割り当てる。
  3. **固定電話の判定**: 上記以外は `companyPhone`（会社の代表番号）に割り当てる。（代表番号が埋まっている場合は `fax` にフォールバックする）
- **重要仕様（改修済）**: FAXと携帯電話が連続する行において、上の行の「FAX」という文脈を下の行が誤って引き継いでしまう不具合を防ぐため、「同じ行のプレフィックスを最優先で評価する」仕様とする。

### 1.4 ノイズ（異常系）の除外処理
- **仕様**: 記号のみの行（例：「-----」「====」）や、文字数が2文字以下の極端に短い行は、ノイズと判定し会社名や氏名として抽出しない（スキップする）。

## 2. セキュリティ・権限管理（Firestore Rules）の詳細仕様

### 2.1 ユーザーアカウントの状態（Status）による制限
- **仕様**: ユーザーの `status` フィールドが `SUSPENDED`（アカウント停止中）の場合、自身が作成した名刺データであっても、**一切の読み取り（get/list）および書き込み（create/update/delete）を拒否**する。

### 2.2 監査ログ（auditLogs）のアクセス制限
- **仕様**: `auditLogs` コレクションは、システム監査のための機密データであるため、以下の制限を設ける。
  - **一般ユーザー**: 読み取り・書き込みともに一切不可（拒否）。
  - **管理者（admin権限）**: 読み取りのみ許可。

### 2.3 リソース保護（他人のデータへのアクセス）
- **仕様**: `businessCards` コレクションにおける更新（update）および削除（delete）は、リソースの所有者（`resource.data.userId == request.auth.uid`）のみに許可し、他人の名刺データを書き換えることは不可とする。
