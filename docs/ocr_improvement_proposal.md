# BizKnot OCR課題と改善提案（アーキテクチャ再定義）

本ドキュメントは、現在のBizKnotシステムにおける名刺読み取り（OCR）の技術的限界と、それを抜本的に解決するための改善案・新しい開発構成を定義するものです。

## 1. 現状のOCRの課題

現在、名刺のテキスト抽出にはフロントエンド（ブラウザ上）で動作する完全無料のライブラリ **`Tesseract.js`** を使用しています。

### 限界と根本原因
*   **意味理解の欠如**: `Tesseract.js` は画像から「文字」を読み取ることはできますが、それが「会社名」なのか「氏名」なのか「部署名」なのかを意味的に判別することは不可能です。
*   **デザインへの非対応**: 日本の名刺はデザインが非常に多様です。正規表現（特定の文字パターンを探すルール）をどれだけ精緻にチューニングしても、少しレイアウトが変わるだけで「氏名」欄に「会社名」が入ってしまうなどの誤認識が必然的に発生します。
*   **結論**: 現在の技術（完全無料のクライアントサイドOCRと正規表現の組み合わせ）では、これ以上の抜本的な精度向上は物理的に不可能です。

## 2. 課題を解決するための提案内容

この課題を解決するためには、単なる「文字認識（OCR）」ではなく、文字の羅列から意味を理解して適切に構造化できる **生成AI（LLM）** や、AIベースの **商用OCR API** に処理を委譲する必要があります。

### 【必須】アーキテクチャの変更
これらの外部AIサービスを利用するためには「APIキー」が必要になります。
APIキーをフロントエンド（HTML/JS）に直接埋め込むと、誰でもキーを盗み出して悪用できてしまうため、**中継地点となるバックエンド（サーバーサイドロジック）を新たに構築することが必須**となります。

*   **推奨構成**: `Firebase Cloud Functions` などのサーバーレス環境を追加。
*   **フロー**: 
    1. フロントエンドから Cloud Functions へ名刺画像を送信
    2. Cloud Functions が安全にAPIキーを使って外部AI（Gemini等）へ画像を送信
    3. 外部AIが「会社名: 〇〇, 氏名: 〇〇」とJSONで結果を返す
    4. Cloud Functions がフロントエンドへ結果を返却

## 3. 改善パターンの比較

新しいアーキテクチャを前提とした場合、利用するAI技術によって無料パターンと有料パターンに分かれます。

### パターンA: 無料で出来るパターン（Gemini API）
Googleの最新生成AIである Gemini（`gemini-1.5-flash` など）の無料枠（AI Studio）を利用するパターンです。

*   **コスト**: 完全無料（※利用制限あり）
*   **メリット**: 無料でありながら「これは会社名、これは氏名」という高度な意味抽出が可能。
*   **デメリット**: 1分間あたりのリクエスト数に上限（例: 15回/分）があり、業務時間中に複数人が一斉に名刺を取り込むとエラー（Rate Limit）になる可能性があります。
*   **適切な開発構成**: `Firebase Cloud Functions` + `Google Gemini API`

### パターンB: 有料・高精度パターン（Google Cloud Vision API / OpenAI）
商用利用を前提としたエンタープライズ向けのAPIを利用するパターンです。

*   **コスト**: 従量課金（1,000枚あたり数百円程度）
*   **メリット**: 利用制限を気にする必要がなく、極めて安定して高速に動作します。特に Google Cloud Vision API の日本語読み取り精度は世界最高クラスであり、OpenAI (`gpt-4o`) と組み合わせることで完璧に近い自動入力が実現します。
*   **デメリット**: クレジットカードの登録が必要で、ランニングコストが発生します。
*   **適切な開発構成**: `Firebase Cloud Functions` + `Google Cloud Vision API` (文字抽出) + `OpenAI API` (意味理解・JSON化)

## 4. 結論と次のアクション

本番稼働において「名刺登録の手間を最小化する（OCR精度を実用レベルにする）」ことを最優先とする場合は、**パターンB（有料パターン）**へのアーキテクチャ移行を強く推奨します。
予算の都合で無料にこだわる場合はパターンAも選択可能ですが、システム全体を支えるためには Firebase 側も Blaze プラン（従量課金ベース、無料枠あり）へのアップグレードが必須となる点にご留意ください。
