# 単体テスト仕様確認メモ

## ① バリデーションについて

現状の詳細設計書に「何文字以内」といった厳密な文字数制限の定義はありません。そのため、基本的には「現在のフロントエンド実装」に依存する形となります。
ただし、以下の最低限のビジネスルールは確定仕様としてテスト観点に含めてください。

*   **必須項目:** 名刺登録/編集時、「会社名・組織名」と「氏名」は必須（空だと保存不可）。
*   **ファイル制限:** アップロードできる画像は「5MB以下の JPG/PNG/JPEG」のみ。
*   **フォーマット:** HTML5の標準属性（type="email", type="tel"）に準じたブラウザ標準のバリデーションに委ねている状態です。

## ② エラーハンドリングについて

すべてのAPI通信エラーの文言定義（メッセージ辞書）まではありませんが、UXに直結する致命的なエラー時の挙動は仕様として定義されていますので、テストに組み込んでください。

*   **OCRタイムアウト（仕様定義 2-3）:** 画像解析が30秒を超過した場合、「通信または画像解析に時間がかかりすぎたためタイムアウトしました」等のアラートを出し、自動的に手動入力画面（business_card_form.html）へ遷移させること。
*   **保存失敗時:** 必須項目漏れやFirestore保存エラー時は、画面内のメッセージエリア（#message-area）に赤字でエラー文言を表示すること。
*   **権限エラー・無効アカウント:** 停止中（SUSPENDED）のアカウントでログインした場合や、未ログインで保護ページへアクセスした場合は、強制的にログイン画面（index.html）へリダイレクトさせること。

## ③ テストデータについて

はい、テスト用の画像やアカウント情報はそちら（テスト実装側）でご準備いただく想定です。
今回のテストを網羅するために、最低限以下のテストデータ・アカウントの準備をお願いします。

### アカウントデータ（Firebase Emulator等に投入）
*   ADMIN 権限を持つ有効なアカウント（@gmail.com または @spps.co.jp）
*   USER 権限を持つ有効なアカウント（同上）
*   Status が SUSPENDED（利用停止）のアカウント
*   許可されていないドメイン（例: @example.com）のアカウント

### ファイルデータ（Playwrightからアップロード用）
*   正常な名刺画像（JPG/PNG）
*   5MBを超えるサイズのダミー画像
*   非対応の拡張子ファイル（PDFやTXTなど）
