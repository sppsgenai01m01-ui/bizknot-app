## 2. 名刺登録機能 (SCR-003, SCR-004, `creation.js`, `form.js`)

| テスト項目ID | テスト観点 | テスト内容 | 期待結果 |
| :--- | :--- | :--- | :--- |
| **REG-001** | **正常系: カメラ起動からのOCR登録** | 「カメラで読み取る」ボタンを押し、OSネイティブカメラで正常な名刺画像（5MB以下）を撮影する。 | OCR処理が実行され、抽出されたテキストと画像プレビューがフォーム画面に表示される。その後、必須項目を埋めて保存すると正常に登録されること。 |
| **REG-002** | **正常系: ファイル選択からのOCR登録** | 「画像を選択する」ボタンを押し、正常な名刺画像（JPG/PNG, 5MB以下）を選択する。 | OCR処理が実行され、抽出されたテキストと画像プレビューがフォーム画面に表示される。その後、必須項目を埋めて保存すると正常に登録されること。 |
| **REG-003** | **正常系: 手動入力** | 「OCRをスキップして手動で入力する」リンクを押下する。 | OCRは実行されず、空のフォーム画面 (`/business_card_form.html`) に遷移すること。 |
| **REG-004** | **正常系: フォームからのデータ保存** | フォーム画面で必須項目（会社名・組織名, 氏名）を含む情報を入力し、「登録する」ボタンを押下する。 | Firestoreにデータが保存され、画面上部のメッセージエリアに「名刺の登録が完了しました」等の成功メッセージが表示されること。 |
| **REG-005** | **正常系: OCR補完（郵便番号）** | 郵便番号が記載された名刺画像を読み込ませる。 | `zipcloud` APIとの連携により、郵便番号に対応する住所が自動的に住所欄に補完入力されること。 |
| **REG-006** | **正常系: OCR補完（役職名）** | 「代表取i締役」など、わずかに誤認識された役職名を含む名刺画像を読み込ませる。 | レーベンシュタイン距離による補正が働き、役職名が「代表取締役」に自動修正されてフォームに表示されること。 |
| **REG-007** | **正常系: 縦書き名刺の認識** | 縦書きでレイアウトされた名刺画像を読み込ませる。 | `jpn+jpn_vert` の設定により、縦書きのテキストがある程度正しく認識・抽出されること。 |
| **REG-008** | **正常系: 登録キャンセル** | OCR後、フォーム画面で何も保存せずに別のページ（例：ダッシュボード）に遷移する。 | `sessionStorage` のデータが破棄され、Firestoreに不要な名刺データが作成されないこと。 |
| **REG-009** | **異常系: ファイルサイズ超過** | 5MBを超える画像をアップロードしようとする。 | アラート等でエラーメッセージ「ファイルサイズが5MBを超えています」が表示され、処理が中断されること。 |
| **REG-010** | **異常系: 非対応ファイル形式** | PDFやTXTなど、JPG/PNG/JPEG以外の形式のファイルをアップロードしようとする。 | アラート等でエラーメッセージ「対応していないファイル形式です」が表示され、処理が中断されること。 |
| **REG-011** | **異常系: OCRタイムアウト** | （テストツール等で擬似的に）画像解析処理に30秒以上かけさせる。 | 「通信または画像解析に時間がかかりすぎたためタイムアウトしました」というアラートが表示され、自動的に手動入力用のフォーム画面に遷移すること。 |
| **REG-012** | **異常系: 保存時バリデーション（必須項目）** | フォーム画面で「会社名・組織名」または「氏名」を空にした状態で「登録する」ボタンを押下する。 | 保存処理は実行されず、画面内のメッセージエリア（`#message-area`）に「必須項目を入力してください」等のエラーメッセージが赤字で表示されること。 |
